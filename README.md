# سكربت النشر التلقائي على Binance Square باستخدام الذكاء الاصطناعي (Groq)

هذا المشروع عبارة عن سكربت Python تلقائي يقوم بجلب أكثر العملات صعوداً وهبوطاً في أسواق العقود الآجلة لـ Binance، وجلب آخر الأخبار الفورية، ثم يرسل هذه البيانات إلى نموذج الذكاء الاصطناعي **Grok أو Llama 3.3 (عبر Groq)** لإنشاء منشورات تحليلية حماسية جذابة باللغة العربية ونشرها تلقائياً بأسلوب "متداول مخضرم" على حسابك في **Binance Square**.

## الميزات:
* **تنوع المحتوى تلقائياً (4 أنواع):**
  * `gainers`: العملات الصاعدة بقوة في العقود الآجلة.
  * `losers`: العملات الهابطة بقوة في العقود الآجلة.
  * `news`: سحب أحدث الأخبار من Cointelegraph RSS وتحليلها.
  * `tips`: نصائح ودروس نفسية وإدارية للمتداولين.
* **أسلوب "المتداول المخضرم":** أسلوب كتابة واقعي وجاد يركز على الانضباط وإدارة المخاطر.
* **الاختيار العشوائي:** يقوم السكربت باختيار نوع المحتوى عشوائياً في كل مرة لضمان تنوع المنشورات.
* **نمطان للجدولة على الاستضافة (Dual Scheduler Modes):**
  * **النمط الدوري (Interval Mode):** النشر كل فترة زمنية ثابتة (مثلاً كل 30 دقيقة).
  * **نمط أوقات الذروة (Peak Hours Mode):** النشر في 10 ساعات محددة يومياً.
* **التباعد البشري العشوائي (Jitter):** تفاوت زمني عشوائي طفيف لمنع كشف السكربت كأداة آلية (Bot).

---

## متطلبات التشغيل:
1. تثبيت لغة **Python (3.9 أو أحدث)**.
2. الحصول على **Groq API Key** من [Groq Console](https://console.groq.com/) أو مفتاح Grok من [xAI Console](https://console.x.ai/).
3. الحصول على **Binance Square API Key** من [Binance Creator Center](https://square.binance.com/creator-center).

---

## طريقة الإعداد والتشغيل:

### 1. تثبيت المكتبات المطلوبة:
```bash
pip install -r requirements.txt
```

### 2. إعداد مفاتيح الوصول:
* قم بنسخ الملف `.env.template` وتسمية النسخة الجديدة بـ `.env`.
* افتح ملف `.env` وقم بوضع مفاتيحك الخاصة وتحديد الإعدادات كالتالي:
```env
BINANCE_SQUARE_API_KEY=مفتاح_بينانس_سكوير_الخاص_بك
GROQ_API_KEY=مفتاح_جروك_أو_جروك_كلاود
GROQ_MODEL=llama-3.3-70b-versatile

# نمط الجدولة: إما 'interval' للنشر الدوري أو 'peak_hours' للنشر في ساعات محددة
SCHEDULER_MODE=interval

# الفاصل الدوري بالنشر (مثال: 1800 ثانية تعادل 30 دقيقة للنشر كل نصف ساعة)
POST_INTERVAL_SECONDS=1800

# أوقات الذروة المحددة (تستخدم فقط في حال SCHEDULER_MODE=peak_hours)
TIMEZONE_OFFSET=1
PEAK_HOURS=08:00,09:30,11:00,13:00,14:30,16:00,18:00,20:00,21:30,23:00

# تباعد عشوائي بشري (True لتفعيله، False لإلغائه)
USE_JITTER=True
JITTER_MAX_SECONDS=180
```

### 3. التشغيل اليدوي أو المعاينة:
* **وضع التجربة والمعاينة (بدون نشر فعلي):**
  ```bash
  python ai_poster.py --dry-run
  ```
* **النشر الفوري لمنشور عشوائي واحد:**
  ```bash
  python ai_poster.py
  ```

## التشغيل والجدولة على الاستضافة أو السيرفر (VPS / Railway):
بمجرد رفع المشروع وربطه بـ **Railway**، سيقرأ السيرفر ملف الـ `Procfile` ويقوم بتشغيل مجدول أوقات الذروة تلقائياً.

### التشغيل في الخلفية على السيرفرات:
```bash
nohup python scheduler.py > scheduler.log 2>&1 &
```
